# 软件测试有哪些前景？
### 作者: 上官人  赞同: 42
感觉一无是处就对了。手工黑盒做三年，这时候一般都是瓶颈期。  
  
有一些人能够感觉到自己的瓶颈，有些人感觉不到。  
  
感觉到自己瓶颈期的人会选择各种各样的突破方式，有的自学一门语言，打算学点儿开发知识；有的学数据库知识，打算往DBA发展；有的学系统学部署，做部署工程师；有的
学习自动化工具，性能的啊，功能的啊，希望变成自动化工程师。  
  
这些是不打算做手工测试了，我们祝他们好运。  
  
继续坚持手工测试呢，有些人就开始转管理了，做组长，做经理，这样；有些人学业务，成为业务专家。  
  
所以其实路很多，看你怎么选择。  
  
那么下面开始泼冷水，让大家清醒一下：  
  
3年测试转开发，意味着什么呢？意味着你从头做起，从初级工程师做起。转其他的岗位也一样，隔行如隔山，基本上都是从初级重头开始做。承认前面3年时间白扔了。  
  
至于3年测试转自动化，那么至少专业技能还留下了。不过你能转的也不过就是测试脚本开发工程师。这个有一定技术，但是说实话，人家给你搭好的框架，你在里边写写画画，
技术水平还不如刚毕业的开发人员。  
  
性能测试到是一条好路，看系统的视角都不一样了。不过性能测试说到底也就是一门手艺，你会发现，所谓性能测试工程师，一般是不参与调优的，真正的大系统是由架构师们去
做调优工作，测试工程师只是帮忙找到瓶颈。  
  
做管理呢，这条路是最简单的啊，全靠忽悠和运气。国内的测试管理那水平是相当的贫富悬殊，所以永远不要怕会饿死。管理水平高的那是什么都有，相当全面；管理水平差的就
是一帮人搭个草台班子，看着好像是那么回事儿，其实狗屁不是。忽悠出来的薪水一点儿不低，嘴皮子好脑袋好使的建议这条路啊。  
  
学业务，这是大多数手工测试工程师会选择的路，国内最赚钱的软件测试，还是做行业的。金融行业，电信行业。银行，基金，保险，支付，这些公司哪个差钱啊！而且这东西只
要你稍微肯用点儿脑子，就能一路混下去。回头新来的业务人员还要跟你打听公司内幕潜规则啥的。  
  
所以，要问测试工程师的前景，有的是，想干啥干啥，看你有没有那个决心和水平了。
