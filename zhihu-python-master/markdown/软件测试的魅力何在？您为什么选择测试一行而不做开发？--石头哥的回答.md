# 软件测试的魅力何在？您为什么选择测试一行而不做开发？
### 作者: 石头哥  赞同: 4
以笔者自己的经历回答下该问题，应该具有一定的普遍性。  
先回答第二个问题吧。  
  
大学毕业后出来找工作（科班出身），当时一心想的是做程序猿，甚至没有听过软件测试这个职位（暴漏年龄了）。面试了几家自己比较有意向的公司，没有成功。然后备受打击
后偶尔看到一家招测试的，也没有多管想着先去试下，结果一下就成功了（尼玛）。  
后面看了测试相关的书，觉得白盒测试貌似很高大上的样子，想着做这个也不错，或者转开发也可以，结果就这样进来了，剩余的大家应该都懂了，后面也会提到。  
接着回答第一个问题  
进入测试后，突然发现自己很享受找bug的过程。每次根据自己的分析发现bug后就很有成就感（这里不是简  
单的执行测试案例），还可以把开发拉过来鄙视一顿（公司对质量很重视，所以不要怀疑，这样的情况还是存在的）。后来，自己想着去尝试自己排查和定位问题，也觉得比写代
码更有趣。这个可能就是我认为的测试的魅力吧！  
不知道有没有回答你的问题？不过还没有完，后面开始推行自动化，测试度量等一堆效率改进后，我就很难找到这样的成就感了。
